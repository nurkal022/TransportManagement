# Дизайн: быстрое переключение аккаунтов + сводка «Все данные»

Дата: 2026-07-17
Проект: transportapp (Flask + Flask-Login + SQLAlchemy, SQLite)

## Цель

1. **Быстрое переключение между аккаунтами без пароля** — под учёткой `admin`
   показывать кнопки с именами учётных записей для мгновенного переключения.
2. **Аккаунт/страница «Все данные»** — сводка-аналитика по маршрутам всех
   пользователей для отчётности (read-only), со скачиванием в Excel.

Обе возможности доступны **только администратору**.

## Контекст (как устроено сейчас)

- `User(UserMixin)`: `id`, `username`, `password`. Роли/флага админа **нет**.
- Данные (`Route`, `Vehicle`, `Driver`, `CargoType`) разграничены по
  `user_id`; каждый запрос — `filter_by(user_id=current_user.id)`.
- Пользователи в базе: `admin` (id 1), `user` (id 2, пусто), `gulim` (id 3).
- `Route` содержит: `date`, `cargo_weight` (т), `fuel` (л), `cargo_volume`,
  `trips_count`, `invoice_paid`, `start_point`/`end_point`, `cargo_type`,
  `vehicle`, `driver`, `user_id`.
- Навбар — Bootstrap 5 (`templates/base.html`).

## Ключевое решение по безопасности

**Без изменения схемы БД.** «Админ» определяется по `username == 'admin'`.
Никаких миграций → нулевой риск для существующих данных. Существующие запросы
`filter_by(user_id=current_user.id)` **не трогаем** — обычные страницы для всех
пользователей работают ровно как раньше.

## Часть 1 — Переключение аккаунтов (имперсонация)

### Логика
- Хелпер `is_admin_context()` → `True`, если реальный владелец сессии — админ:
  `current_user.username == 'admin'` **или** в сессии стоит `admin_id`,
  указывающий на пользователя `admin`.
- Context processor прокидывает `is_admin_context` и список пользователей в
  шаблоны (для навбара).

### Маршрут `GET /switch-account/<int:user_id>` (`@login_required`)
1. Если `not is_admin_context()` → `abort(403)`.
2. `target = User.query.get_or_404(user_id)`.
3. Запомнить настоящего админа: `session['admin_id'] = <id админа>` (если ещё не
   стоит — берём из текущего админского `current_user.id`).
4. `login_user(target)`.
5. Если `target.username == 'admin'` → удалить `session['admin_id']`
   (вернулись в обычный админский режим).
6. `flash("Вы вошли как <username>")`, redirect на `request.referrer или index`.

### Навбар (только в админ-контексте)
Выпадающий пункт «👤 <текущий username>» с кнопкой-ссылкой на каждый аккаунт
(`/switch-account/<id>`); текущий подсвечен (`active`). Пункт виден, только если
`is_admin_context` — обычные пользователи (gulim/user) его не видят.

### Выход
`logout` дополнительно очищает `session['admin_id']`.

## Часть 2 — Страница «Все данные» (сводка, read-only)

### Маршрут `GET /summary` (`@login_required`)
1. Если `not is_admin_context()` → `abort(403)`.
2. Опциональный фильтр периода: query-параметры `date_from`, `date_to`
   (по `Route.date`). Пустые → без ограничения.
3. Агрегация по **всем** пользователям (без фильтра по `user_id`).

### Показатели
- **Итого:** маршрутов, суммарно рейсов (`trips_count`), вес груза (т),
  топливо (л), объём.
- **Счета:** оплачено / не оплачено (кол-во и доля).
- **По аккаунтам:** для каждого `user` — маршруты, рейсы, вес, топливо.
- **По типам груза:** топ по кол-ву/весу.
- **По машинам:** топ по кол-ву маршрутов/весу.
- **Помесячно:** кол-во маршрутов и вес по месяцам (`date`).

Страница только читает данные. Создание/редактирование/удаление отсутствует.

### Шаблон
Новый `templates/summary.html` (Bootstrap 5, как остальные страницы): карточки
с итогами, таблицы разбивок, поля фильтра дат, кнопка «Скачать Excel».

### Экспорт `GET /summary/export.xlsx` (`@login_required`, админ-контекст)
`openpyxl` (уже в зависимостях). Те же данные и фильтр периода, что и на
странице: листы «Итоги», «По аккаунтам», «По типам груза», «По машинам»,
«Помесячно». Отдаётся как файл `summary_<date_from>_<date_to>.xlsx`.

### Навбар
Ссылка «📊 Все данные» → `/summary` (только в админ-контексте).

## Что НЕ ломаем (границы)

- Схема БД без изменений, миграций нет.
- Существующие маршруты и запросы с `user_id` не меняются.
- Все новые эндпоинты под серверной проверкой `is_admin_context()`.
- Не-админ ничего нового не видит и получает 403 при прямом заходе.

## Крайние случаи

- `/switch-account/<id>` на несуществующего пользователя → 404.
- Не-админ на `/switch-account` или `/summary`/`/summary/export.xlsx` → 403.
- Имперсонация: сессия помнит настоящего админа (`admin_id`), поэтому
  переключатель не пропадает и можно вернуться в `admin`.
- `logout` очищает `admin_id`.
- Пустой период фильтра → сводка по всем датам.

## Тестирование и деплой

- Разработка и проверка — локально, на копии базы (`instance/transport.db`),
  сервер не затрагивается.
- Ручная проверка сценариев: вход admin → переключение на gulim → назад;
  не-админ не видит кнопок и получает 403; сводка совпадает с суммами по
  отдельным аккаунтам; Excel выгружается и открывается.
- Деплой на сервер только после проверки: бэкап кода+базы → загрузка изменённых
  файлов → `systemctl restart transportapp` → проверка, что сайт отвечает.
